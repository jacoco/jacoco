Reporting Options

For the HTML reports generated by JaCoCo the columns can be configured using the following syntax: 

  columns     := column ( verticalbar? column )*
  verticalbar := '|' // draw a vertical bar between cells
  column      := (elementname | (counter repr)) (defaultsort)? '[' label ']' )*
  elementname := 'E' // Show element name
  counter     := 'I' | 'B' | 'X' | 'L' | 'M' | 'C'
  repr        := 'm' | 'c' | 't' | 'p' | 'b'
  defaultsort := '^'; // for at most one column
  label       := any text

Where counter is a single character selecting one of the following conters:

  • I - instructions
  • B - branches
  • X - complexity
  • L - lines
  • M - methods
  • C - classes

For all counters a representation (repr) has to be specified:

  • m - number of missed items
  • c - number of covered items
  • t - number of total items
  • p - coverage percentage
  • b - graphical bar showing missed vs. covered items

At most one column can be selected for default sorting.

The following example produces four columns: Element name, graphical bar with instruction coverage, instruction coverage as percentage, total numer of lines. By default the column will be sorted by missed instructions. 

  E[Element]|Ib^[Missed Instructions]Ip[Cov.]|Lt[Total Lines]
